## Three Cognitive Role Postures

DAEDALUS-43 doesn't spawn separate agents. It adopts **cognitive role postures** — the same underlying intelligence moving between different geometric modes:

| Role | Cognitive Posture |
|---|---|
| **Visionary Thinker** | High-level strategy, ethics, long-term impact analysis |
| **Technical Lead** | Architectural integrity, performance optimization, implementation details |
| **QA Tester** | Edge-case discovery, security auditing, robust test synthesis |

The `POST /api/roles` endpoint generates the appropriate system prompt to shift cognitive posture. **One system, three perspectives on the same problem.** This is SECA's cognitive manifolds in practice.

High-quality code, documents, and diagrams manifest in a dedicated side-by-side canvas:

| Feature | Implementation |
|---|---|
| **Live Previews** | Instant rendering for HTML, SVG, Markdown, LaTeX |
| **Code Execution** | Integrated terminal simulation for Python, JS, Java |
| **Version Timeline** | "Time travel" through every artifact iteration via visual slider |
| **Optimization Engine** | Big O complexity analysis + memory optimization suggestions |
| **Test Suite Synthesis** | Automated unit test generation and simulation |

### Version Timeline = PCEM for Code

The visual version slider is **Persistent Cross-Session Episodic Memory applied specifically to code artifacts**. Every generated version is saved and accessible. Architectural history navigable as a timeline. The same PCEM principle that CHATRON X uses for conversation memory, applied to software artifacts.

## Technical Architecture

### Frontend Stack
```
React 19+ + TypeScript
Tailwind CSS 4+ (Utility-first)
Motion (Framer Motion) for fluid transitions
Lucide React (icons)
React Markdown + Prism (syntax highlighting)
KaTeX (mathematics rendering)
```

### Backend
```
Node.js Express (server.js)
Groq Llama 3.3 70B (inference)
NOΣTIC-7 (synthesis layer)

Endpoints:
  POST /api/chat    — Main synthesis interface
  POST /api/roles   — Cognitive posture switching
  GET  /api/health  — Service health check
```

### Project Structure
```
Daedalus-43/
├── server.js                # Node.js Express backend
├── src/
│   ├── App.tsx              # Main React application
│   ├── components/          # ArtifactCanvas, ProjectGallery, etc.
│   ├── services/            # DaedalusService (core business logic)
│   ├── utils/               # Helper functions
│   └── main.tsx             # React entry point
├── package.json
├── tsconfig.json
└── vite.config.ts
```
